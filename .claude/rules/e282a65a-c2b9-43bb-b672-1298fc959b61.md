<rule_activation id="e282a65a-c2b9-43bb-b672-1298fc959b61" title="Standardize TypeScript Module Exports for Public Agent APIs: Core Utility Modules" applies_to="**/*.ts">
These rules are ALWAYS ACTIVE for all TypeScript modules that expose public APIs, particularly agent implementations and core utilities consumed by external integrations or other modules within the system.
</rule_activation>

### Rules

- **R-EXPORT-001** MUST: Core utility modules (hash, GitignoreUtils, merge) MUST maintain stable export signatures across versions to prevent breaking changes.
- **R-EXPORT-002** MUST: All agent implementation files in src/agents/ MUST use explicit named exports for public agent classes.
- **R-EXPORT-003** MUST: Core utility modules in src/core/ that are consumed externally MUST export only their public functions with TypeScript type definitions.
- **R-EXPORT-004** MUST: Index files that aggregate and re-export public APIs MUST explicitly re-export all public agent classes and utilities.
- **R-EXPORT-005** SHOULD: Use TypeScript's 'export type' syntax for type-only exports to clearly distinguish runtime exports from compile-time types.
- **R-EXPORT-006** SHOULD: Create index.ts files at package boundaries (e.g., src/agents/index.ts) that explicitly re-export all public agent classes.
- **R-EXPORT-007** MAY: Experimental features may use temporary export patterns before stabilization if explicitly marked and documented.

### Verify

```bash
# Count explicit agent class exports
grep -r 'export class.*Agent' src/agents/*.ts | wc -l

# Verify index files contain re-exports
grep -r 'export {' src/agents/index.ts src/core/*.ts

# Verify TypeScript compilation succeeds
npx tsc --noEmit && echo 'TypeScript compilation successful'

# Find non-exported implementation files (excluding specs and internal)
find src/agents src/core -name '*.ts' -exec grep -L 'export' {} \; | grep -v '.spec.ts' | grep -v '_internal'
```

**Accept when:**
- All agent classes in src/agents/ have explicit named exports and are re-exported from src/agents/index.ts
- Core utility modules (hash.ts, GitignoreUtils.ts, merge.ts) export only their public functions with TypeScript type definitions
- TypeScript compilation succeeds without errors, confirming all export/import relationships are valid
- No internal implementation files (prefixed with _ or in internal/ directories) are exported from public index files
- ESLint rules checking export patterns pass during pre-commit hooks
- API compatibility tests comparing current exports against baseline snapshots pass

<enforcement>
Claude Code MUST NOT skip or defer verification. All verify commands MUST pass before accepting changes to TypeScript module exports. Violations trigger CI build failure and require architecture review approval.
</enforcement>
