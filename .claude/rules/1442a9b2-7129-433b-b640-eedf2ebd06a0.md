<rule_activation id="1442a9b2-7129-433b-b640-eedf2ebd06a0" title="Adopt Vitest with vi.mock() for Unit Testing TypeScript Agent Modules: Unit Test Files" applies_to="tests/unit/**/*.test.ts">
These rules are ALWAYS ACTIVE for all unit test development in the codebase, particularly for TypeScript agent modules and CLI handlers.
</rule_activation>

### Rules

- **R-VITEST-001** MUST: Unit test files MUST follow the naming convention `*.test.ts` and be located in the `tests/unit` directory structure.
- **R-VITEST-002** MUST: All unit test files in `tests/unit/` MUST use Vitest imports (`import { describe, test, expect, vi } from 'vitest'`).
- **R-VITEST-003** MUST: Agent test files SHOULD use `vi.mock()` for dependency isolation, with at least 80% of agent test files using this pattern.
- **R-VITEST-004** MUST: Test suites MUST implement `beforeEach(() => vi.clearAllMocks())` to prevent mock state leakage between tests.
- **R-VITEST-005** MUST: Module-level mocks using `vi.mock('module-path')` MUST be placed at the top of test files before imports to ensure proper hoisting.
- **R-VITEST-006** SHOULD: External dependencies (file system, network calls, third-party libraries) in agent tests SHOULD be mocked consistently using `vi.mock()`.
- **R-VITEST-007** SHOULD: Test files with approved exceptions SHOULD include exception reference (EX-XXX) in file headers.

### Verify

```bash
# Count vi.mock usage in unit tests
grep -r "vi.mock" tests/unit/ | wc -l

# Verify Vitest imports are present
grep -r "from 'vitest'" tests/unit/**/*.test.ts | wc -l

# Count test files following naming convention
find tests/unit -name '*.test.ts' -type f | wc -l

# Sample test structure verification
grep -r "describe\|test\|it" tests/unit/ | head -5

# Run Vitest suite
vitest run
```

**Accept when:**
- All unit test files in `tests/unit/` use Vitest imports (`import { describe, test, expect, vi } from 'vitest'`)
- At least 80% of agent test files use `vi.mock()` for dependency isolation
- Test files follow naming convention `*.test.ts` and are located in `tests/unit` directory structure
- CI pipeline successfully runs `vitest` command and all unit tests pass
- Mock cleanup is implemented with `beforeEach(() => vi.clearAllMocks())` or equivalent in test suites
- Module-level mocks are placed at the top of test files before imports

Claude Code MUST NOT skip or defer verification. All unit test files must be validated against these rules before acceptance. Violations must be flagged and remediated according to the enforcement process: code review blocking, CI pipeline failure, and automated PR comments with ADR reference.
