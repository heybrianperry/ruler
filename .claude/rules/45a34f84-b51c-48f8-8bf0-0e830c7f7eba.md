<rule_activation id="45a34f84-b51c-48f8-8bf0-0e830c7f7eba" title="Adopt Unified Configuration Loading with Input Validation: Configuration Validation Include" applies_to="**/*">
These rules are ALWAYS ACTIVE for all configuration loading operations across agent implementations, core engine components, and test suites. All configuration sources MUST be processed through the unified configuration loader with mandatory input validation.
</rule_activation>

### Rules

- **R-UCL-001** MUST: Configuration validation MUST include type checking, range validation, and format verification appropriate to each configuration parameter.
- **R-UCL-002** MUST: All agent implementations (ZedAgent, GeminiCliAgent, RooCodeAgent, OpenCodeAgent, QwenCodeAgent) MUST use UnifiedConfigLoader for configuration loading.
- **R-UCL-003** MUST: Core engine components (apply-engine, configuration loaders) MUST use UnifiedConfigLoader for configuration loading.
- **R-UCL-004** MUST: MCP propagation modules MUST use UnifiedConfigLoader for configuration loading.
- **R-UCL-005** MUST: Test suites that exercise configuration loading MUST use UnifiedConfigLoader.
- **R-UCL-006** MUST: No direct TOML parsing or environment variable access outside of UnifiedConfigLoader in production code.
- **R-UCL-007** SHOULD: Configuration validation errors SHOULD provide clear, actionable error messages.
- **R-UCL-008** MAY: Legacy components during migration phase may temporarily use direct configuration parsing (EXC-001).
- **R-UCL-009** MAY: Performance-critical paths where configuration is loaded once at startup and cached may defer to UnifiedConfigLoader (EXC-002).

### Verify

```bash
# Count UnifiedConfigLoader usage
grep -r 'UnifiedConfigLoader' src/ | wc -l

# Verify all agent implementations use UnifiedConfigLoader
grep -r 'new UnifiedConfigLoader\|import.*UnifiedConfigLoader' src/agents/ src/core/ src/mcp/

# Run configuration validation tests
npm test -- --grep 'configuration.*validation'

# Enforce no direct config parser imports in production
eslint src/ --rule 'no-restricted-imports: [error, {patterns: ["**/direct-config-parser"]}]'
```

**Accept when:**
- All agent implementations (ZedAgent, GeminiCliAgent, RooCodeAgent, OpenCodeAgent, QwenCodeAgent) import and use UnifiedConfigLoader
- Configuration validation tests pass for both valid inputs and properly reject invalid inputs with clear error messages
- No direct TOML parsing or environment variable access occurs outside of UnifiedConfigLoader in production code
- Code review confirms all new configuration loading code uses the unified approach
- ESLint rules prevent direct configuration parsing imports
- Integration tests validate configuration loading behavior across all components
