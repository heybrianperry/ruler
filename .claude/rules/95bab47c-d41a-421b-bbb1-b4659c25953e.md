<rule_activation id="95bab47c-d41a-421b-bbb1-b4659c25953e" title="Standardize Configuration Loading with Unified Environment Management: Configuration Loaders Support" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent implementations, configuration loaders, and runtime environment management components. All code that loads configuration, manages environment variables, or initializes runtime settings MUST comply with these rules.
</rule_activation>

### Rules

- **R-CONFIG-001** MUST: Configuration loaders MUST support multiple configuration sources with a defined precedence order (e.g., environment variables override file-based config).
- **R-CONFIG-002** MUST: All agent implementations (OpenCodeAgent, MistralVibeAgent, ZedAgent, CodexCliAgent, AmazonQCliAgent, GeminiCliAgent, FirebenderAgent, RooCodeAgent) MUST load configuration exclusively through UnifiedConfigLoader or approved abstraction layer.
- **R-CONFIG-003** MUST: Configuration values MUST be validated and sanitized before use to prevent injection attacks and runtime errors.
- **R-CONFIG-004** MUST: All configuration schemas MUST be defined with type-safe interfaces and validation logic.
- **R-CONFIG-005** MUST: Sensitive configuration values MUST be redacted in debug logging.
- **R-CONFIG-006** SHOULD: Configuration source precedence SHOULD follow: CLI args > environment variables > config files > defaults.
- **R-CONFIG-007** SHOULD: Implement caching for validated configuration values to optimize performance in high-frequency access scenarios.

### Verify

```bash
# Count direct process.env accesses outside UnifiedConfigLoader
grep -r 'process\.env\[' src/ --exclude-dir=node_modules | grep -v UnifiedConfigLoader | wc -l

# Run configuration validation tests
npm run test:config -- --grep 'configuration validation'

# Check for direct process.env access violations via ESLint
eslint src/ --rule 'no-restricted-syntax: [error, {selector: MemberExpression[object.object.name=process][object.property.name=env], message: Use UnifiedConfigLoader instead of direct process.env access}]'
```

**Accept when:**
- All agent implementations load configuration exclusively through UnifiedConfigLoader or approved abstraction layer
- Configuration validation tests pass for all supported configuration sources (TOML, environment variables, VSCode settings, MCP)
- No direct process.env accesses exist outside of the UnifiedConfigLoader implementation itself
- All configuration schemas are defined with type-safe interfaces and validation logic
- Sensitive configuration values are redacted in all logging output
- Configuration loading behavior is validated through CI pipeline integration tests
