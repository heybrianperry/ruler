These rules are ALWAYS ACTIVE for all configuration loading operations across agent implementations, core engine components, and test suites. All configuration sources MUST be processed through the unified configuration loader with mandatory input validation.
</rule_activation>

### Rules

- **R-UCL-001** MUST: All agent implementations (ZedAgent, GeminiCliAgent, RooCodeAgent, OpenCodeAgent, QwenCodeAgent) import and use UnifiedConfigLoader for configuration loading.
- **R-UCL-002** MUST: All configuration sources (TOML files, environment variables, runtime parameters) be processed through UnifiedConfigLoader with mandatory input validation.
- **R-UCL-003** MUST: No direct TOML parsing or environment variable access occur outside of UnifiedConfigLoader in production code.
- **R-UCL-004** MAY: Components MAY extend the UnifiedConfigLoader with domain-specific validation rules when necessary.
- **R-UCL-005** MUST: Configuration validation tests pass for both valid inputs and properly reject invalid inputs with clear error messages.
- **R-UCL-006** SHOULD: Core engine components (apply-engine, configuration loaders) and MCP propagation modules use UnifiedConfigLoader.
- **R-UCL-007** SHOULD: Test suites that exercise configuration loading (test_manual_toml.js, test_toml_options.js) validate both valid and invalid configuration scenarios.
