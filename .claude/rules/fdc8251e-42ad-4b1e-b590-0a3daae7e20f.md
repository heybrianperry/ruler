<rule_activation id="fdc8251e-42ad-4b1e-b590-0a3daae7e20f" title="Adopt TypeScript Test Files with .test.ts Extension as Standard Testing Convention" applies_to="**/*.test.ts">
These rules are ALWAYS ACTIVE for all TypeScript test file creation and organization within the codebase.
</rule_activation>

### Rules

- **R-TS-TEST-001** MUST NOT: Test files MUST NOT be placed in the same directory as production source code.
- **R-TS-TEST-002** MUST: All TypeScript test files MUST use the .test.ts extension for consistent naming convention.
- **R-TS-TEST-003** MUST: Test files MUST be organized into distinct directories (tests/unit, tests/integration, tests/e2e) demonstrating clear separation of test types.
- **R-TS-TEST-004** MUST: Test runner configuration MUST include glob patterns that match **/*.test.ts files for automatic discovery.
- **R-TS-TEST-005** SHOULD: Configure tsconfig.json to include tests/ directory and ensure test files are type-checked alongside production code.
- **R-TS-TEST-006** SHOULD: Implement ESLint rules or custom linting to enforce test file naming conventions and directory structure.
- **R-TS-TEST-007** SHOULD: Document the testing structure in CONTRIBUTING.md or TESTING.md with examples of proper test file creation and organization.

### Verify

```bash
# Verify all test files follow .test.ts convention
find tests/ -type f -name '*.ts' ! -name '*.test.ts' | grep -v 'helpers\|fixtures\|mocks' && echo 'Found non-test TypeScript files in tests/' || echo 'All test files follow .test.ts convention'

# Verify test runner is configured for .test.ts pattern
grep -r 'testMatch\|testRegex' jest.config.* tsconfig.json package.json | grep -q '\.test\.ts' && echo 'Test runner configured for .test.ts pattern' || echo 'Warning: Test runner may not be configured correctly'

# Verify test directory structure exists
ls tests/unit/ tests/integration/ tests/e2e/ 2>/dev/null && echo 'Test directory structure exists' || echo 'Warning: Expected test directories not found'

# Verify no test files exist in production source directories
find src/ -name '*.test.ts' 2>/dev/null && echo 'Found test files in production source directories' || echo 'No test files in production source directories'
```

**Accept when:**
- All TypeScript test files in the codebase use the .test.ts extension and are located under the tests/ directory
- Test runner configuration includes glob patterns that match **/*.test.ts files
- CI/CD pipeline successfully discovers and executes all test files without manual configuration of individual test paths
- No TypeScript test files exist in production source directories outside of tests/
- Test directory structure with unit/, integration/, and e2e/ subdirectories is established and maintained

<enforcement>
Claude Code MUST verify test file naming conventions and directory structure before accepting TypeScript test file creation or organization. Violations of R-TS-TEST-001 and R-TS-TEST-002 MUST be flagged and corrected. Verification is mandatory and MUST NOT be skipped or deferred.
</enforcement>
