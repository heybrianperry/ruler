<rule_activation id="3f757ee0-3163-4246-a3ff-3d0de1aef14f" title="Standardize Unit Testing Framework for Agent Components: Integration Tests Placed" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent component development and testing activities within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** MAY: Integration tests MAY be placed in separate directories outside of `tests/unit/` to distinguish from unit tests.
- **R-TEST-002** MUST: All test files follow the naming pattern `*.test.ts` and are located in appropriate subdirectories under `tests/unit/`.
- **R-TEST-003** MUST: Test suite executes successfully with at least 80% code coverage across agent components.
- **R-TEST-004** MUST: At least 6 agent test files exist in `tests/unit/agents/` directory, maintaining the established pattern.
- **R-TEST-005** SHOULD: New agent implementations include corresponding unit tests in the correct location following established conventions.
- **R-TEST-006** SHOULD: Test files for CLI handlers and command-line interface logic be placed in `tests/unit/cli/` subdirectory.
- **R-TEST-007** SHOULD: Invalid input scenarios and error handling be explicitly tested as part of unit test coverage.

### Verify

```bash
# Verify test file naming and location conventions
find tests/unit -name '*.test.ts' | grep -E 'tests/unit/(agents|cli)/.+\.test\.ts$'

# Verify code coverage meets minimum threshold
npm test -- --coverage --coverageThreshold='{"global":{"lines":80}}'

# Verify minimum number of agent test files exist
ls tests/unit/agents/*.test.ts | wc -l | grep -E '^[6-9]|^[1-9][0-9]+$'
```

**Accept when:**
- All test files follow the naming pattern `*.test.ts` and are located in appropriate subdirectories under `tests/unit/`
- Test suite executes successfully with at least 80% code coverage across agent components
- At least 6 agent test files exist in `tests/unit/agents/` directory, maintaining the established pattern
- New agent implementations include corresponding unit tests in `tests/unit/agents/` with proper naming convention
- CLI handler tests are located in `tests/unit/cli/` subdirectory
