<rule_activation id="d13a40f3-89ba-49a5-859f-b52b4d73284b" title="Standardize Environment Variable Configuration Management Across Agent Implementations: Configuration State Immutable" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent implementations, core configuration components, MCP propagation components, and test fixtures that interact with environment variable configuration management.
</rule_activation>

### Rules

- **R-CONFIG-001** MUST: Configuration state MUST be immutable after initial load to prevent race conditions in concurrent execution contexts.
- **R-CONFIG-002** MUST: All agent implementations (ZedAgent, GeminiCliAgent, RooCodeAgent, OpenCodeAgent, QwenCodeAgent, and future agents) MUST use UnifiedConfigLoader for environment variable access.
- **R-CONFIG-003** MUST: No direct `process.env` access is permitted in agent implementations outside of the UnifiedConfigLoader component.
- **R-CONFIG-004** MUST: Configuration validation MUST occur at startup with fail-fast behavior to catch configuration errors before runtime execution.
- **R-CONFIG-005** SHOULD: Configuration access SHOULD be abstracted through TypeScript interfaces to provide type safety and IDE autocomplete.
- **R-CONFIG-006** SHOULD: All required and optional environment variables SHOULD be documented in a central README or configuration guide with examples for each environment type.

### Verify

```bash
# Verify no direct process.env access in agent implementations
grep -r 'process\.env' src/agents/ src/core/ --include='*.ts' | grep -v 'UnifiedConfigLoader' | wc -l | grep -q '^0$'

# Verify all 8+ agent files reference UnifiedConfigLoader
grep -r 'UnifiedConfigLoader' src/agents/ --include='*.ts' | wc -l | awk '{if($1>=8) exit 0; else exit 1}'

# Verify configuration validation tests achieve 80% coverage
npm test -- --testPathPattern='config.*test' --coverage --coverageThreshold='{"global":{"branches":80,"functions":80,"lines":80}}'
```

**Accept when:**
- No direct `process.env` access exists in agent implementations outside of UnifiedConfigLoader
- All 8+ agent files reference UnifiedConfigLoader for configuration access
- Configuration validation tests achieve at least 80% code coverage and include failure scenarios
- Configuration immutability is enforced through TypeScript const assertions or Object.freeze() patterns
- All configuration parameters are validated at load time with clear error messages
