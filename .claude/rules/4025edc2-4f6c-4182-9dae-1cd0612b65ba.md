<rule_activation id="4025edc2-4f6c-4182-9dae-1cd0612b65ba" title="Standardize on Vitest for Unit Testing Agent Modules: Test Files Use" applies_to="tests/unit/agents/**/*.test.ts">
These rules are ALWAYS ACTIVE for all agent module unit tests in tests/unit/agents/. This ADR standardizes testing framework and TypeScript usage across all agent implementations.
</rule_activation>

### Rules

- **R-VITEST-001** MUST: Test files MUST use TypeScript (.ts extension) to maintain type safety in test code
- **R-VITEST-002** MUST: All agent unit tests in tests/unit/agents/ MUST import test functions from 'vitest' package
- **R-VITEST-003** MUST: Test files MUST follow naming convention {AgentName}.test.ts
- **R-VITEST-004** MUST: Agent unit tests MUST NOT use alternative testing frameworks (jest, mocha, ava) in the tests/unit/agents/ directory
- **R-VITEST-005** MUST: CI pipeline MUST fail if agent tests are found using non-Vitest frameworks

### Verify

```bash
# Verify all agent test files import from vitest
grep -r "from 'vitest'" tests/unit/agents/*.test.ts | wc -l

# Count total agent test files
find tests/unit/agents -name '*.test.ts' -type f | wc -l

# Run all agent tests with vitest
npx vitest run tests/unit/agents/ --reporter=verbose

# Check for non-Vitest framework usage
grep -r "from 'jest'\|from 'mocha'\|from 'ava'" tests/unit/agents/*.test.ts | wc -l
```

**Accept when:**
- All agent test files in tests/unit/agents/ import test functions from 'vitest' package
- Test files follow naming convention {AgentName}.test.ts and execute successfully with vitest runner
- No agent unit tests use alternative testing frameworks (jest, mocha, ava) in the tests/unit/agents/ directory
- All .test.ts files in tests/unit/agents/ are written in TypeScript

<enforcement>
Claude Code MUST verify all rules before accepting agent test files. Violations block pull requests until corrected. Exceptions require documented justification and engineering lead approval.
</enforcement>
