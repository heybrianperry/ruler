<rule_activation id="18c735f8-8361-4730-ad66-b17482fed1fc" title="Adopt Structured Test Organization with Layered Test Suites: Projects Introduce Additional" applies_to="**/*.test.ts">
These rules are ALWAYS ACTIVE for all test development and CI/CD pipeline configuration.
</rule_activation>

### Rules

- **R-TSO-001** MAY: Projects MAY introduce additional test layers (e.g., contract, performance, security) as needed for specific domains.
- **R-TSO-002** MUST: All test files be located within `tests/unit/`, `tests/integration/`, or `tests/e2e/` directories.
- **R-TSO-003** MUST: Unit tests contain no external I/O operations (network, database, file system writes).
- **R-TSO-004** MUST: CI/CD pipeline configuration include separate stages or jobs for each test layer.
- **R-TSO-005** MUST: Test execution reports clearly identify which layer each test belongs to.
- **R-TSO-006** SHOULD: Use consistent naming convention with `.test.ts` extensions for all test files.
- **R-TSO-007** SHOULD: Configure test runner (jest.config.js, vitest.config.ts) to support different settings per layer (timeouts, setup files).

### Verify

```bash
# Verify test files are in correct directory structure
find tests/ -type f -name '*.test.ts' | grep -E '^tests/(unit|integration|e2e)/' | wc -l

# Check for I/O operations in unit tests
grep -r 'describe\|test\|it' tests/unit/ | grep -E '(fetch|http|database|fs\.write)' && echo 'Found I/O in unit tests' || echo 'Unit tests clean'

# Verify test directories exist
test -d tests/unit && test -d tests/integration && test -d tests/e2e && echo 'Test directories exist' || echo 'Missing test directories'
```

**Accept when:**
- All test files are located within `tests/unit/`, `tests/integration/`, or `tests/e2e/` directories
- Unit tests contain no external I/O operations (network, database, file system writes)
- CI/CD pipeline configuration includes separate stages or jobs for each test layer
- Test execution reports clearly identify which layer each test belongs to
- Test files follow consistent naming convention with `.test.ts` extensions

<enforcement>
Claude Code MUST verify test organization structure before accepting test files. Violations of R-TSO-002, R-TSO-003, R-TSO-004, and R-TSO-005 MUST be flagged. Automated CI/CD checks MUST fail if tests are found outside approved directory structure or if unit tests contain I/O operations.
</enforcement>
