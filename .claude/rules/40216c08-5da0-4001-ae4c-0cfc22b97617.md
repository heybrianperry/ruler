These rules are ALWAYS ACTIVE for all unit test development in the codebase, particularly for TypeScript agent modules and CLI handlers.

### Rules

- **R-VITEST-001** MUST: Use Vitest as the testing framework for all TypeScript unit tests in tests/unit directory.
- **R-VITEST-002** MUST: Use vi.mock() at the top of test files before imports to ensure proper hoisting for module-level mocking.
- **R-VITEST-003** MUST: Import test utilities from 'vitest': `import { describe, test, expect, vi } from 'vitest'`
- **R-VITEST-004** MUST: Implement beforeEach(() => vi.clearAllMocks()) in test suites to prevent mock state leakage between tests.
- **R-VITEST-005** MAY: Tests MAY use vi.spyOn() for partial mocking when only specific methods need to be intercepted.
- **R-VITEST-006** SHOULD: Follow naming convention *.test.ts for all unit test files located in tests/unit directory structure.
- **R-VITEST-007** SHOULD: Mock external dependencies like file system, network calls, and third-party libraries consistently across agent tests.
- **R-VITEST-008** MUST: Configure vitest.config.ts with appropriate test file patterns matching tests/unit/**/*.test.ts

### Verify

```bash
# Count vi.mock usage in unit tests
grep -r "vi.mock" tests/unit/ | wc -l

# Verify Vitest imports are present
grep -r "from 'vitest'" tests/unit/**/*.test.ts | wc -l

# Count total test files
find tests/unit -name '*.test.ts' -type f | wc -l

# Sample test structure verification
grep -r "describe\|test\|it" tests/unit/ | head -5
```

**Accept when:**
- All unit test files in tests/unit/ use Vitest imports (import { describe, test, expect, vi } from 'vitest')
- At least 80% of agent test files use vi.mock() for dependency isolation
- Test files follow naming convention *.test.ts and are located in tests/unit directory structure
- CI pipeline successfully runs vitest command and all unit tests pass
- beforeEach hooks with vi.clearAllMocks() are present in test suites
