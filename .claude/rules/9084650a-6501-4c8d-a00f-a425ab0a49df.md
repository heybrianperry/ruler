<rule_activation id="9084650a-6501-4c8d-a00f-a425ab0a49df" title="Standardize Cache Layer Configuration Management Pattern: Configuration Changes Logged" applies_to="**/*">
These rules are ALWAYS ACTIVE for all runtime environment configuration and cache layer implementations. All configuration management code MUST comply with these rules.
</rule_activation>

### Rules

- **R-CACHE-001** SHOULD: Configuration changes SHOULD be logged to facilitate debugging and audit trails.

### Scope

In scope:
- All cache layer configuration in MCP propagation modules
- All cache-related settings in VSCode extension configuration
- Runtime environment configuration affecting cache behavior
- Configuration propagation between application boundaries

Out of scope:
- Cache implementation internals (eviction policies, storage mechanisms)
- Non-cache related configuration management
- Third-party library configuration that doesn't interact with the cache layer
- Build-time or compile-time configuration

### Verify

```bash
# Verify cache configuration pattern is present
grep -r 'cache.*config' src/ --include='*.ts' | grep -E '(propagate|settings)' || echo 'Pattern not found'

# Find cache configuration files
find src/ -name '*settings*.ts' -o -name '*propagate*.ts' | xargs grep -l 'cache' || echo 'No cache configuration files found'

# Verify cache configuration tests exist
npm test -- --grep 'cache.*configuration' || echo 'No cache configuration tests found'
```

**Accept when:**
- All cache layer configuration code follows the standardized pattern (IR signature 3233fb2b7f24bb1c444af07faab034c3)
- Configuration propagation between MCP contexts and settings is consistent and validated
- Unit tests verify cache configuration behavior across all runtime contexts
- Configuration changes are logged for debugging and audit trail purposes

<enforcement>
Claude Code MUST NOT skip or defer verification. Automated code review checks for cache configuration patterns in pull requests are mandatory. CI pipeline verification commands must detect deviations from the standard pattern. Architecture review is required for any new cache layer implementations. Pull requests with non-compliant cache configuration patterns are blocked until corrected.
</enforcement>
