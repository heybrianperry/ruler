### Rules

- **R-TSM-001** SHOULD: Test suites SHOULD maintain consistent structure and testing patterns across all agent implementations to facilitate maintenance.

### Verify

```bash
# Verify test files follow naming pattern and are in correct subdirectories
find tests/unit -name '*.test.ts' | grep -E 'tests/unit/(agents|cli)/.+\.test\.ts$'

# Verify code coverage meets minimum threshold of 80%
npm test -- --coverage --coverageThreshold='{"global":{"lines":80}}'

# Verify at least 6 agent test files exist in tests/unit/agents/
ls tests/unit/agents/*.test.ts | wc -l | grep -E '^[6-9]|^[1-9][0-9]+$'
```

**Accept when:**
- All test files follow the naming pattern `*.test.ts` and are located in appropriate subdirectories under `tests/unit/`
- Test suite executes successfully with at least 80% code coverage across agent components
- At least 6 agent test files exist in `tests/unit/agents/` directory, maintaining the established pattern
- All agent component implementations (AmpAgent, WarpAgent, JulesAgent, WindsurfAgent, OpenCodeAgent, JunieAgent, and future agents) have corresponding unit tests
- CLI handler components have unit tests validating command-line interface behavior
- Error handling and invalid input scenarios are tested

<enforcement>
Claude Code MUST verify test file naming conventions, directory structure, and code coverage thresholds before approving changes to agent components or CLI handlers. Verification is mandatory and MUST NOT be skipped or deferred.
</enforcement>
