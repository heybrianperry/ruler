<rule_activation id="ba5b8dfd-f245-4513-b6c4-622e62f23c35" title="Standardize Test File Organization in tests/ Directory with Unit and Integration Separation: Test Files Use" applies_to="**/*">
These rules are ALWAYS ACTIVE for all test file creation and organization within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** MUST: Test files MUST use the naming convention *.test.ts or *.test.js to be discoverable by test runners
- **R-TEST-002** MUST: All test files with .test.ts or .test.js extension MUST be located under tests/unit/ or tests/integration/ directories
- **R-TEST-003** MUST: Test files MUST NOT be co-located with source files in src/ directories
- **R-TEST-004** SHOULD: Unit tests SHOULD mirror source code structure within tests/unit/ for intuitive test location discovery
- **R-TEST-005** SHOULD: Integration tests SHOULD be organized within tests/integration/ for multi-component interaction testing
- **R-TEST-006** MAY: Test fixtures, mock data, and helper functions MAY be organized separately in tests/helpers/ or tests/utils/

### Verify

```bash
# Check for test files outside unit/integration directories
find tests/ -name '*.test.ts' -o -name '*.test.js' | grep -v -E '^tests/(unit|integration)/' && echo 'Found tests outside unit/integration dirs' || echo 'All tests properly organized'

# Verify required test directories exist
test -d tests/unit && test -d tests/integration && echo 'Required test directories exist' || echo 'Missing required test directories'

# Count unit tests
npm test -- --testPathPattern=tests/unit --listTests | wc -l

# Verify test runner can execute unit and integration tests separately
npm test -- --testPathPattern=tests/unit
npm test -- --testPathPattern=tests/integration
```

**Accept when:**
- All test files with .test.ts or .test.js extension are located under tests/unit/ or tests/integration/ directories
- Test runner can successfully execute unit tests and integration tests as separate suites
- CI/CD pipeline successfully runs unit tests independently from integration tests with appropriate timing thresholds
- No test files are found outside the approved directory structure
- Required test directories (tests/unit/ and tests/integration/) exist and are properly configured

<enforcement>
Claude Code MUST NOT skip or defer verification. All test files MUST conform to the naming convention and directory structure. CI/CD pipeline MUST fail if test files are found outside tests/unit/ or tests/integration/ directories. Code review MUST flag incorrectly placed tests for revision.
</enforcement>
