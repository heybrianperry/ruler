<rule_activation id="1a54fe6c-848f-4ee3-82ea-886f1be5561e" title="Standardize Public API Export Patterns for Agent Integration: Public Exports Use" applies_to="src/agents/**/*.ts,src/core/**/*.ts,src/mcp/**/*.ts">
These rules are ALWAYS ACTIVE for all agent implementation files, core utility modules, integration modules, and index files that provide public API surfaces.
</rule_activation>

### Rules

- **R-API-001** MUST: Public API exports MUST use explicit named exports rather than default exports to improve IDE support and refactoring safety.

### Verify

```bash
# Verify no default exports in agent implementation files (excluding test files)
grep -r 'export default' src/agents/*.ts | grep -v '.test.ts' | wc -l | grep -q '^0$'

# Verify all agent classes follow naming convention with 'Agent' suffix
grep -r 'export class.*Agent' src/agents/*.ts | wc -l

# Verify central index file exists and contains exports for all agent implementations
test -f src/agents/index.ts && grep -q 'export.*Agent' src/agents/index.ts
```

**Accept when:**
- No default exports found in agent implementation files (excluding test files)
- All agent classes follow the naming convention with 'Agent' suffix
- Central index file exists and contains exports for all agent implementations
- All public API modules use explicit named exports
- All agents are registered in the central index file

<enforcement>
Claude Code MUST verify all conditions in the **Accept when** section before approving changes to agent implementations, core utilities, integration modules, or index files. Verification is mandatory and MUST NOT be skipped or deferred.
</enforcement>
