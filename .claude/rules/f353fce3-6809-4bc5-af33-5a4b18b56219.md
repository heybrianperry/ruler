These rules are ALWAYS ACTIVE for all agent component development and testing activities.
</rule_activation>

### Rules

- **R-AGENT-001** MAY: Teams MAY organize agent tests into subdirectories within tests/unit/agents/ if the number of agents grows significantly

### Verify

```bash
# Verify agent test directory structure exists
find tests/unit/agents -name '*.test.ts' | wc -l

# List all agent test files
ls tests/unit/agents/*.test.ts 2>/dev/null || echo 'No agent tests found'

# Confirm agent test directory exists
test -d tests/unit/agents && echo 'Agent test directory exists' || echo 'Missing agent test directory'
```

**Accept when:**
- The tests/unit/agents/ directory exists and contains .test.ts files
- Each agent implementation class has a corresponding {AgentName}.test.ts file in tests/unit/agents/
- Test files can be discovered and executed by the project's test runner without additional configuration

<enforcement>
Claude Code MUST verify agent test structure compliance. CI pipeline MUST fail if an agent class is detected without a corresponding unit test file. Pull requests MUST be blocked from merging if test structure violations are detected. Exceptions require documentation in TESTING_EXCEPTIONS.md with technical lead approval and remediation plan.
</enforcement>
