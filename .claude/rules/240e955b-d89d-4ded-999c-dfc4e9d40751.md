<rule_activation id="240e955b-d89d-4ded-999c-dfc4e9d40751" title="Adopt TypeScript Test Files with .test.ts Extension as Standard Testing Convention: Integration Tests Placed" applies_to="**/*.test.ts">
These rules are ALWAYS ACTIVE for all TypeScript test file creation and organization within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** SHOULD: Integration tests SHOULD be placed in tests/integration/ directory and focus on component interactions.
- **R-TEST-002** MUST: All TypeScript test files MUST use the .test.ts extension.
- **R-TEST-003** MUST: Test files MUST be located under the tests/ directory with type-based subdirectories (tests/unit/, tests/integration/, tests/e2e/).
- **R-TEST-004** MUST: Test runner configuration MUST include glob patterns that match **/*.test.ts files.
- **R-TEST-005** SHOULD: Test files SHOULD NOT exist in production source directories outside of tests/.
- **R-TEST-006** SHOULD: tsconfig.json SHOULD include tests/ directory and ensure test files are type-checked alongside production code.
- **R-TEST-007** MAY: ESLint rules or custom linting MAY be used to enforce test file naming conventions and directory structure.

### Verify

```bash
# Verify all test files follow .test.ts convention
find tests/ -type f -name '*.ts' ! -name '*.test.ts' | grep -v 'helpers\|fixtures\|mocks' && echo 'Found non-test TypeScript files in tests/' || echo 'All test files follow .test.ts convention'

# Verify test runner is configured for .test.ts pattern
grep -r 'testMatch\|testRegex' jest.config.* tsconfig.json package.json | grep -q '\.test\.ts' && echo 'Test runner configured for .test.ts pattern' || echo 'Warning: Test runner may not be configured correctly'

# Verify test directory structure exists
ls tests/unit/ tests/integration/ tests/e2e/ 2>/dev/null && echo 'Test directory structure exists' || echo 'Warning: Expected test directories not found'
```

**Accept when:**
- All TypeScript test files in the codebase use the .test.ts extension and are located under the tests/ directory
- Test runner configuration includes glob patterns that match **/*.test.ts files
- CI/CD pipeline successfully discovers and executes all test files without manual configuration of individual test paths
- No TypeScript test files exist in production source directories outside of tests/
- Integration tests are placed in tests/integration/ directory and focus on component interactions
