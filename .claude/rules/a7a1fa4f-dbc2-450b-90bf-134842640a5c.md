<rule_activation id="a7a1fa4f-dbc2-450b-90bf-134842640a5c" title="Standardize Test File Organization for CI/CD Pipeline Validation: Mcp Related Tests" applies_to="**/*">
These rules are ALWAYS ACTIVE for all test files within the CI/CD pipeline and apply to all agent-related test suites, configuration validation tests, and integration tests.
</rule_activation>

### Rules

- **R-MCP-001** SHOULD: MCP-related tests SHOULD validate key propagation, remote configuration loading, and provider-specific behaviors (Claude, Gemini, Firebase).
- **R-TEST-002** SHOULD: Use test file templates that include standard sections for input validation, configuration override testing, and nested behavior validation.
- **R-TEST-003** SHOULD: Establish naming conventions: `tests/unit/agents/{AgentName}.test.ts` for unit tests, `tests/integration/{feature-name}.test.ts` for integration tests, and `tests/{config-aspect}.test.ts` for configuration validation.
- **R-TEST-004** SHOULD: Implement test helpers for common validation patterns (MCP config validation, HTTP type checking, agent-specific disable logic) to reduce duplication.
- **R-TEST-005** SHOULD: Configure CI/CD pipeline to run unit tests first for fast feedback, followed by integration tests, with clear failure reporting that maps to the test organization structure.

### Verify

```bash
# Count test files matching the expected organization pattern
find tests/ -name '*.test.ts' | grep -E '(unit/agents/|integration/|claude-|gemini-|mcp-)' | wc -l

# Verify agent test files exist in unit/agents directory
grep -r 'describe.*Agent' tests/unit/agents/ | wc -l

# Verify security-critical input validation tests exist
grep -r 'input.*validation\|type.*field\|MCP.*config' tests/ | wc -l
```

**Accept when:**
- All agent implementations have corresponding test files in `tests/unit/agents/` directory
- Configuration validation tests exist for all security-critical input fields (type, HTTP config, MCP settings)
- Test organization follows the three-tier structure: unit tests, integration tests, and configuration tests with clear naming conventions
- MCP-related tests validate key propagation, remote configuration loading, and provider-specific behaviors

<enforcement>
Claude Code MUST NOT skip or defer verification. CI/CD pipeline blocks merge if required tests are missing or failing. Code review process flags missing test coverage for new agents or configuration changes. Automated alerts are sent to team leads when test coverage drops below thresholds.
</enforcement>
