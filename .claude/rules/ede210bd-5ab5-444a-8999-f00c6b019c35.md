### Rules

- **R-ENV-001** MUST: All agent implementations MUST load configuration through the UnifiedConfigLoader or equivalent centralized configuration management component.

### Verify

```bash
# Verify no direct process.env access in agent implementations outside UnifiedConfigLoader
grep -r 'process\.env' src/agents/ src/core/ --include='*.ts' | grep -v 'UnifiedConfigLoader' | wc -l | grep -q '^0$'

# Verify all 8+ agent files reference UnifiedConfigLoader for configuration access
grep -r 'UnifiedConfigLoader' src/agents/ --include='*.ts' | wc -l | awk '{if($1>=8) exit 0; else exit 1}'

# Verify configuration validation tests achieve at least 80% code coverage
npm test -- --testPathPattern='config.*test' --coverage --coverageThreshold='{"global":{"branches":80,"functions":80,"lines":80}}'
```

**Accept when:**
- No direct process.env access exists in agent implementations outside of UnifiedConfigLoader
- All 8+ agent files reference UnifiedConfigLoader for configuration access
- Configuration validation tests achieve at least 80% code coverage and include failure scenarios

<enforcement>
Claude Code MUST NOT skip or defer verification. All three verification commands MUST pass before accepting configuration management changes in agent implementations.
</enforcement>
