<rule_activation id="c1d43c47-60b2-4478-a3cd-950ea5e33985" title="Adopt Vitest with vi.mock() for Unit Testing TypeScript Agent Modules: Each Agent Implementation" applies_to="tests/unit/**/*.test.ts">
These rules are ALWAYS ACTIVE for all unit test development in the codebase, particularly for agent implementations (AmpAgent, WarpAgent, JulesAgent, WindsurfAgent, OpenCodeAgent, JunieAgent) and CLI handler tests.
</rule_activation>

### Rules

- **R-VITEST-001** MUST: Each agent implementation MUST have a corresponding unit test file that mocks external dependencies using Vitest's vi.mock() API.
- **R-VITEST-002** MUST: All unit test files in tests/unit/ MUST use Vitest imports (import { describe, test, expect, vi } from 'vitest').
- **R-VITEST-003** MUST: Test files MUST follow naming convention *.test.ts and be located in tests/unit directory structure.
- **R-VITEST-004** MUST: Use vi.mock('module-path') at the top of test files before imports to ensure proper hoisting.
- **R-VITEST-005** MUST: Implement beforeEach(() => vi.clearAllMocks()) in test suites to prevent mock state leakage between tests.
- **R-VITEST-006** SHOULD: For agent tests, mock external dependencies like file system, network calls, and third-party libraries consistently.
- **R-VITEST-007** SHOULD: Complement unit tests with integration tests that use real dependencies to mitigate over-mocking risks.

### Verify

```bash
# Count vi.mock usage in unit tests
grep -r "vi.mock" tests/unit/ | wc -l

# Verify Vitest imports are present
grep -r "from 'vitest'" tests/unit/**/*.test.ts | wc -l

# Count total test files
find tests/unit -name '*.test.ts' -type f | wc -l

# Sample test structure
grep -r "describe\|test\|it" tests/unit/ | head -5
```

**Accept when:**
- All unit test files in tests/unit/ use Vitest imports (import { describe, test, expect, vi } from 'vitest')
- At least 80% of agent test files use vi.mock() for dependency isolation
- Test files follow naming convention *.test.ts and are located in tests/unit directory structure
- CI pipeline successfully runs vitest command and all unit tests pass
