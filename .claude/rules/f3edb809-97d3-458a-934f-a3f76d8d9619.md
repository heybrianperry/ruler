<rule_activation id="f3edb809-97d3-458a-934f-a3f76d8d9619" title="Standardize Test File Organization in tests/ Directory with Unit and Integration Separation: Unit Test Files" applies_to="**/*">
These rules are ALWAYS ACTIVE for all test file creation and organization within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** SHOULD: Unit test files SHOULD mirror the directory structure of the source files they test (e.g., src/core/RuleProcessor.ts → tests/unit/core/RuleProcessor.test.ts)

### Verify

```bash
# Verify all test files are in unit or integration directories
find tests/ -name '*.test.ts' -o -name '*.test.js' | grep -v -E '^tests/(unit|integration)/' && echo 'Found tests outside unit/integration dirs' || echo 'All tests properly organized'

# Verify required test directories exist
test -d tests/unit && test -d tests/integration && echo 'Required test directories exist' || echo 'Missing required test directories'

# Count unit tests
npm test -- --testPathPattern=tests/unit --listTests | wc -l
```

**Accept when:**
- All test files with .test.ts or .test.js extension are located under tests/unit/ or tests/integration/ directories
- Test runner can successfully execute unit tests and integration tests as separate suites
- CI/CD pipeline successfully runs unit tests independently from integration tests with appropriate timing thresholds

<enforcement>
Claude Code MUST NOT skip or defer verification. All test files MUST comply with the directory structure defined in R-TEST-001. Violations MUST be flagged during code review and CI pipeline checks MUST fail if test files are found outside tests/unit/ or tests/integration/ directories.
</enforcement>
