<rule_activation id="61b0b307-1fd6-4fc9-a3df-d4c7671dffda" title="Adopt Unit Testing for CI/CD Pipeline Components: Teams Supplement Unit" applies_to="**/*">
These rules are ALWAYS ACTIVE for all CI/CD pipeline components, build tooling, and deployment automation code.
</rule_activation>

### Rules

- **R-CICD-001** MAY: Teams MAY supplement unit tests with integration tests for end-to-end pipeline validation, but unit tests remain mandatory.
- **R-CICD-002** MUST: Organize unit tests under tests/unit/ directory mirroring the source structure, using .test.ts extension for TypeScript projects.
- **R-CICD-003** MUST: Use test fixtures and mocks to isolate CI/CD components from external dependencies (file systems, network, external services).
- **R-CICD-004** SHOULD: Focus tests on validating configuration parsing, error handling, path resolution, and business logic rather than framework behavior.
- **R-CICD-005** MUST: Run unit tests in pre-commit hooks and CI pipeline to catch regressions before code review.
- **R-CICD-006** SHOULD: Maintain test execution time under 30 seconds for the full unit test suite to enable rapid feedback.
- **R-CICD-007** SHOULD: Document complex test scenarios with comments explaining the edge case or regression being prevented.

### Verify

```bash
# Count unit test files
find tests/unit -name '*.test.ts' -type f | wc -l

# Count test cases
grep -r "describe\|it\|test" tests/unit/ --include='*.test.ts' | wc -l

# Execute unit test suite
npm test -- --testPathPattern=tests/unit --passWithNoTests=false 2>&1 | grep -E '(PASS|FAIL|Tests:)'
```

**Accept when:**
- Unit test files exist under tests/unit/ directory with .test.ts extension for all CI/CD agent implementations and configuration loaders
- Test suite executes successfully with passing tests covering core functionality, error handling, and edge cases
- CI pipeline includes unit test execution as a required quality gate that must pass before merge
- All CI/CD components (FirebaseAgent, CodexCliAgent, ConfigLoader, etc.) have corresponding unit tests
- Tests validate configuration parsing, error handling, path resolution, and business logic

<enforcement>
Claude Code MUST NOT skip or defer verification. Unit tests for CI/CD components are mandatory before merge. Pull requests without unit tests for new CI/CD components MUST be blocked.
</enforcement>
