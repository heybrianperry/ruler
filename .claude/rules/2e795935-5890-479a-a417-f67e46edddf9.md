<rule_activation id="2e795935-5890-479a-a417-f67e46edddf9" title="Standardize Test File Organization in tests/ Directory with Unit and Integration Separation: Integration Tests Organized" applies_to="**/*">
These rules are ALWAYS ACTIVE for all test file creation and organization within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** MUST: Integration tests MUST be organized under tests/integration/ subdirectory
- **R-TEST-002** MUST: All test files with .test.ts or .test.js extension MUST be located under tests/unit/ or tests/integration/ directories
- **R-TEST-003** SHOULD: Unit tests for core engine components, agents, and utilities SHOULD mirror source code structure within tests/unit/
- **R-TEST-004** SHOULD: Test runners SHOULD be configured to recognize the tests/ directory structure and support running unit and integration tests separately
- **R-TEST-005** MAY: Test fixtures, mock data files, and test utilities MAY be organized separately in tests/helpers/ or tests/utils/

### Verify

```bash
# Check for test files outside unit/integration directories
find tests/ -name '*.test.ts' -o -name '*.test.js' | grep -v -E '^tests/(unit|integration)/' && echo 'Found tests outside unit/integration dirs' || echo 'All tests properly organized'

# Verify required test directories exist
test -d tests/unit && test -d tests/integration && echo 'Required test directories exist' || echo 'Missing required test directories'

# Count unit tests
npm test -- --testPathPattern=tests/unit --listTests | wc -l
```

**Accept when:**
- All test files with .test.ts or .test.js extension are located under tests/unit/ or tests/integration/ directories
- Test runner can successfully execute unit tests and integration tests as separate suites
- CI/CD pipeline successfully runs unit tests independently from integration tests with appropriate timing thresholds
- Required test directories (tests/unit/ and tests/integration/) exist and are properly configured

<enforcement>
Claude Code MUST NOT skip or defer verification. All test files MUST comply with the organizational structure defined in this rule. Violations detected by automated CI checks MUST be resolved before merge.
</enforcement>
