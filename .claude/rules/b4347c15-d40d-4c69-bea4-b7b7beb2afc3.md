<rule_activation id="b4347c15-d40d-4c69-bea4-b7b7beb2afc3" title="Adopt Unified Configuration Loading Pattern for Runtime Environment Management: Components Cache Loaded" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent implementations and configuration management components within the system. All new agents and configuration loaders MUST comply with these rules.
</rule_activation>

### Rules

- **R-CONFIG-001** MAY: Components MAY cache loaded configuration data to avoid redundant file I/O operations, provided cache invalidation is properly handled.
- **R-CONFIG-002** MUST: All agent implementations (ZedAgent, GeminiCliAgent, RooCodeAgent, OpenCodeAgent, QwenCodeAgent, and future agents) use UnifiedConfigLoader for configuration loading.
- **R-CONFIG-003** MUST: Core configuration management components (UnifiedConfigLoader, apply-engine) implement the unified configuration loading pattern.
- **R-CONFIG-004** MUST: MCP propagation and integration components that require configuration access use UnifiedConfigLoader.
- **R-CONFIG-005** MUST: Import UnifiedConfigLoader at the entry point of your application or agent and load configuration before instantiating agents.
- **R-CONFIG-006** MUST: Define TypeScript interfaces for configuration structure and use them to type configuration objects returned by UnifiedConfigLoader.
- **R-CONFIG-007** SHOULD: Use environment variable prefixes (e.g., APP_) to avoid conflicts with system environment variables.
- **R-CONFIG-008** SHOULD: Provide a configuration validation command that can be run independently to check TOML files before deployment.
- **R-CONFIG-009** SHOULD: When adding new configuration options, update the TOML schema validation rules and add test cases covering the new options.

### Verify

```bash
# Verify UnifiedConfigLoader usage across agent implementations
grep -r "UnifiedConfigLoader" src/agents/ | wc -l

# Verify imports of UnifiedConfigLoader
grep -r "import.*UnifiedConfigLoader" src/ --include="*.ts" --include="*.js"

# Verify configuration loading tests pass
npm test -- --grep "config.*load" 2>&1 | grep -E "(passing|failing)"
```

**Accept when:**
- All agent implementations import and use UnifiedConfigLoader for configuration loading
- Configuration loading tests pass successfully, including tests for TOML parsing, environment variable precedence, and validation
- No direct file system reads for configuration files exist outside of UnifiedConfigLoader implementation

<enforcement>
Claude Code MUST NOT skip or defer verification. Code review process checks that new agents use UnifiedConfigLoader. CI pipeline runs configuration loading tests on every commit. Static analysis tools flag direct configuration file reads outside UnifiedConfigLoader. Pull requests that bypass UnifiedConfigLoader are rejected during code review.
</enforcement>
