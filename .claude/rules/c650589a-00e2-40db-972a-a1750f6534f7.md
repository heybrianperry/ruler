### Rules

- **R-TS-001** SHOULD: Negative test cases (testing invalid inputs or error conditions) SHOULD be included alongside positive test cases to ensure comprehensive coverage.

### Verify

```bash
# Verify all test files use .test.ts extension
find tests/unit -name '*.test.ts' -type f | wc -l

# Verify test files contain test patterns
grep -r "describe\|test\|it" tests/unit/**/*.test.ts | head -n 5

# Verify TypeScript compilation succeeds for test files
npx tsc --noEmit --project tsconfig.test.json 2>&1 | grep -c 'error' || echo '0'
```

**Accept when:**
- All test files in tests/unit/ directory use the .test.ts extension and are discoverable by the test runner
- TypeScript compilation succeeds for all test files without type errors
- Test runner configuration correctly identifies and executes all .test.ts files, with no orphaned or undiscovered tests
- Negative test cases are present alongside positive test cases in test files

<enforcement>
Claude Code MUST NOT skip or defer verification. All test files MUST follow the .test.ts naming convention, be properly organized in tests/unit/ hierarchy, and include both positive and negative test cases. Violations block CI builds and pull requests until remediated.
</enforcement>
