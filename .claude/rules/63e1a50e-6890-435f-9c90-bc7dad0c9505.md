### Rules

- **R-AGENT-001** MUST: Agent unit test files MUST be placed in tests/unit/agents/ directory with the naming pattern {AgentName}.test.ts
- **R-AGENT-002** MUST: All agent test files in tests/unit/agents/ MUST import test functions from 'vitest' package
- **R-AGENT-003** MUST: Agent unit tests MUST NOT use alternative testing frameworks (jest, mocha, ava) in the tests/unit/agents/ directory
- **R-AGENT-004** SHOULD: Create shared test utilities in tests/unit/agents/helpers/ for common agent mocking and setup patterns
- **R-AGENT-005** SHOULD: Configure CI pipeline to run vitest with coverage reporting and fail builds on coverage threshold violations

### Verify

```bash
# Verify vitest imports in agent test files
grep -r "from 'vitest'" tests/unit/agents/*.test.ts | wc -l

# Count total agent test files
find tests/unit/agents -name '*.test.ts' -type f | wc -l

# Run vitest on agent tests with verbose reporting
npx vitest run tests/unit/agents/ --reporter=verbose

# Check for non-vitest framework usage
grep -r "from 'jest'\|from 'mocha'\|from 'ava'" tests/unit/agents/*.test.ts | wc -l
```

**Accept when:**
- All agent test files in tests/unit/agents/ import test functions from 'vitest' package
- Test files follow naming convention {AgentName}.test.ts and execute successfully with vitest runner
- No agent unit tests use alternative testing frameworks (jest, mocha, ava) in the tests/unit/agents/ directory
- vitest is installed as a dev dependency with appropriate TypeScript configuration in vitest.config.ts
