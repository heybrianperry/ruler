<rule_activation id="e3e05666-7088-42a2-a354-d593bfe4c690" title="Standardize Unit Testing Framework for Agent Components: Test Files Use" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent component development and testing activities within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** MUST: Test files MUST use the `.test.ts` extension to be recognized by the testing framework.

### Verify

```bash
# Verify all test files follow the .test.ts naming convention and are in correct directories
find tests/unit -name '*.test.ts' | grep -E 'tests/unit/(agents|cli)/.+\.test\.ts$'

# Verify test suite executes with minimum 80% code coverage
npm test -- --coverage --coverageThreshold='{"global":{"lines":80}}'

# Verify at least 6 agent test files exist in tests/unit/agents/
ls tests/unit/agents/*.test.ts | wc -l | grep -E '^[6-9]|^[1-9][0-9]+$'
```

**Accept when:**
- All test files follow the naming pattern `*.test.ts` and are located in appropriate subdirectories under `tests/unit/`
- Test suite executes successfully with at least 80% code coverage across agent components
- At least 6 agent test files exist in `tests/unit/agents/` directory, maintaining the established pattern
- All agent component implementations (AmpAgent, WarpAgent, JulesAgent, WindsurfAgent, OpenCodeAgent, JunieAgent, and future agents) have corresponding unit tests
- CLI handler components have corresponding unit tests in `tests/unit/cli/`

<enforcement>
Claude Code MUST NOT skip or defer verification. All pull requests adding new agents or CLI handlers without corresponding unit tests in the correct location will fail CI checks and be blocked from merging. Code coverage drops below 80% threshold will fail the build and require additional tests before merge approval.
</enforcement>
