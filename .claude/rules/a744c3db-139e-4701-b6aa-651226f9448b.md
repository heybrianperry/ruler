<rule_activation id="a744c3db-139e-4701-b6aa-651226f9448b" title="Adopt Unified Configuration Loading Pattern for Runtime Environment Management: Toml Configuration Files" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent implementations and configuration management components within the system. All new agents and configuration loaders MUST comply with these rules.
</rule_activation>

### Rules

- **R-TOML-001** MUST: TOML configuration files MUST be validated against a schema before being applied to ensure type safety and prevent runtime errors.
- **R-TOML-002** MUST: All agent implementations (ZedAgent, GeminiCliAgent, RooCodeAgent, OpenCodeAgent, QwenCodeAgent, and future agents) MUST use UnifiedConfigLoader for configuration loading.
- **R-TOML-003** MUST: Core configuration management components (UnifiedConfigLoader, apply-engine) MUST implement unified configuration loading patterns.
- **R-TOML-004** MUST: MCP propagation and integration components that require configuration access MUST use UnifiedConfigLoader.
- **R-TOML-005** SHOULD: Import UnifiedConfigLoader at the entry point of your application or agent and load configuration before instantiating agents.
- **R-TOML-006** SHOULD: Define TypeScript interfaces for your configuration structure and use them to type the configuration objects returned by UnifiedConfigLoader.
- **R-TOML-007** SHOULD: When adding new configuration options, update the TOML schema validation rules and add test cases covering the new options.
- **R-TOML-008** SHOULD: Use environment variable prefixes (e.g., APP_) to avoid conflicts with system environment variables.
- **R-TOML-009** MAY: Provide a configuration validation command that can be run independently to check TOML files before deployment.

### Exceptions

- **EXC-001**: A component requires configuration before UnifiedConfigLoader is initialized (e.g., bootstrap configuration).
- **EXC-002**: Performance-critical paths where configuration loading overhead is measured and documented as unacceptable.

### Verify

```bash
# Verify UnifiedConfigLoader usage across agent implementations
grep -r "UnifiedConfigLoader" src/agents/ | wc -l

# Verify imports of UnifiedConfigLoader
grep -r "import.*UnifiedConfigLoader" src/ --include="*.ts" --include="*.js"

# Verify configuration loading tests pass
npm test -- --grep "config.*load" 2>&1 | grep -E "(passing|failing)"
```

**Accept when:**
- All agent implementations import and use UnifiedConfigLoader for configuration loading
- Configuration loading tests pass successfully, including tests for TOML parsing, environment variable precedence, and validation
- No direct file system reads for configuration files exist outside of UnifiedConfigLoader implementation
- TOML configuration files are validated against a schema before being applied

<enforcement>
Claude Code MUST NOT skip or defer verification. Code review process checks that new agents use UnifiedConfigLoader. CI pipeline runs configuration loading tests on every commit. Static analysis tools flag direct configuration file reads outside UnifiedConfigLoader. Pull requests that bypass UnifiedConfigLoader are rejected during code review.
</enforcement>
