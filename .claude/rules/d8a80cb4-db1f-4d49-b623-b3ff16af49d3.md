### Rules

- **R-TEST-001** SHOULD: Test files SHOULD use a Jest-compatible or Vitest-compatible testing framework to ensure broad ecosystem support and familiar syntax for developers.
- **R-TEST-002** MUST: All TypeScript unit test files in the codebase MUST use the `.test.ts` file extension for consistent test discovery and execution.
- **R-TEST-003** MUST: Test files MUST be organized in a parallel directory structure under `tests/unit/` with agent-specific tests in `tests/unit/agents/` and CLI tests in `tests/unit/cli/`.
- **R-TEST-004** MUST: Test files MUST be written in TypeScript to enable compile-time type checking and provide better IDE support.
- **R-TEST-005** SHOULD: Test suites SHOULD include both positive test cases and negative test cases (e.g., invalid input scenarios) to enable comprehensive test coverage.

### Verify

```bash
# Count all .test.ts files in tests/unit directory
find tests/unit -name '*.test.ts' -type f | wc -l

# Verify test structure contains describe/test/it blocks
grep -r "describe\|test\|it" tests/unit/**/*.test.ts | head -n 5

# Verify TypeScript compilation succeeds for all test files
npx tsc --noEmit --project tsconfig.test.json 2>&1 | grep -c 'error' || echo '0'
```

**Accept when:**
- All test files in `tests/unit/` directory use the `.test.ts` extension and are discoverable by the test runner
- TypeScript compilation succeeds for all test files without type errors
- Test runner configuration correctly identifies and executes all `.test.ts` files, with no orphaned or undiscovered tests
- Test files follow the directory hierarchy: `tests/unit/agents/` for agent tests and `tests/unit/cli/` for CLI handler tests
