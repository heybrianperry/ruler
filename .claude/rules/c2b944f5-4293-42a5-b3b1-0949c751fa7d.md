<rule_activation id="c2b944f5-4293-42a5-b3b1-0949c751fa7d" title="Standardize Test File Naming and Organization for CI/CD Integration: Pipelines Configured Discover" applies_to="**/*.test.ts">
These rules are ALWAYS ACTIVE for all test file creation and CI/CD pipeline configuration activities.
</rule_activation>

### Rules

- **R-TEST-001** MUST: CI/CD pipelines MUST be configured to discover and execute all files matching the `**/*.test.ts` glob pattern.
- **R-TEST-002** MUST: All TypeScript test files in the project MUST use the `.test.ts` extension.
- **R-TEST-003** SHOULD: Test files SHOULD be organized into appropriate directories (unit/integration/e2e) based on their scope.
- **R-TEST-004** SHOULD: Test organization SHOULD separate concerns by test type to enable selective test execution.

### Verify

```bash
# Count test files matching the standard pattern
find tests -name '*.test.ts' | wc -l

# Verify test file content structure
grep -r 'describe\|test\|it' tests/ --include='*.test.ts' | head -n 5

# Check test directory organization
ls -la tests/unit tests/integration tests/e2e 2>/dev/null || echo 'Test directories not found'

# Verify no non-compliant test files exist
find tests -name '*.spec.ts' -o -name '*test.js' -o -name '*spec.js' | wc -l
```

**Accept when:**
- All test files use the `.test.ts` extension and are discoverable by the configured test runner
- Test files are organized into appropriate directories (unit/integration/e2e) based on their scope
- CI/CD pipeline successfully discovers and executes all tests without manual configuration for new test files
- Test coverage reports accurately reflect all test files in the project
- No test files matching alternative patterns (`.spec.ts`, `.test.js`, etc.) exist in the codebase

<enforcement>
Claude Code MUST NOT skip or defer verification of test file naming patterns and CI/CD pipeline discovery configuration. Violations MUST be flagged during code review and pre-commit validation.
</enforcement>
