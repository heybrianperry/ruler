<rule_activation id="a858c5ae-14b4-4900-ba42-7882b097256c" title="Standardize Test File Organization in tests/ Directory with Unit and Integration Separation: Additional Test Subdirectories" applies_to="**/*">
These rules are ALWAYS ACTIVE for all test file creation and organization within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** MUST: Organize all TypeScript and JavaScript test files under `tests/unit/` or `tests/integration/` directories based on test type.
- **R-TEST-002** MUST: Place unit tests for core engine components, agents, and utilities under `tests/unit/`.
- **R-TEST-003** MUST: Place integration tests for multi-component interactions under `tests/integration/`.
- **R-TEST-004** MAY: Create additional test subdirectories (e.g., `tests/e2e/`, `tests/performance/`) for specialized test types following the same organizational principles.
- **R-TEST-005** MUST: Ensure test runner can execute unit tests and integration tests as separate suites.
- **R-TEST-006** MUST: Configure CI/CD pipeline to run unit tests on every commit and integration tests on pull requests or specific branches.
- **R-TEST-007** SHOULD: Mirror source code structure in unit tests to make test locations intuitive and ensure comprehensive coverage tracking.
- **R-TEST-008** MUST NOT: Co-locate test files with source files (e.g., `src/core/RuleProcessor.test.ts` alongside `RuleProcessor.ts`).
- **R-TEST-009** MUST NOT: Use flat test directory structure without unit/integration separation.

### Verify

```bash
# Verify all tests are properly organized under unit or integration directories
find tests/ -name '*.test.ts' -o -name '*.test.js' | grep -v -E '^tests/(unit|integration)/' && echo 'Found tests outside unit/integration dirs' || echo 'All tests properly organized'

# Verify required test directories exist
test -d tests/unit && test -d tests/integration && echo 'Required test directories exist' || echo 'Missing required test directories'

# Count unit tests
npm test -- --testPathPattern=tests/unit --listTests | wc -l
```

**Accept when:**
- All test files with `.test.ts` or `.test.js` extension are located under `tests/unit/` or `tests/integration/` directories
- Test runner can successfully execute unit tests and integration tests as separate suites
- CI/CD pipeline successfully runs unit tests independently from integration tests with appropriate timing thresholds
- No test files exist outside the approved directory structure
- Required `tests/unit/` and `tests/integration/` directories are present

<enforcement>
Claude Code MUST verify test file organization against these rules. Violations detected by automated CI checks MUST be flagged. Pre-commit hooks MUST validate test file locations. Pull requests with incorrectly placed tests MUST be rejected or flagged for revision.
</enforcement>
