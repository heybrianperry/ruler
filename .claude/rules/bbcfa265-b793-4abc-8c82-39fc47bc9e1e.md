### Rules

- **R-TS-TEST-001** MUST: All unit test files use the `.test.ts` naming convention.
- **R-TS-TEST-002** MUST: Test files be organized in a structured directory hierarchy under `tests/unit/`, with agent-specific tests in `tests/unit/agents/` and CLI tests in `tests/unit/cli/`.
- **R-TS-TEST-003** MUST: TypeScript test files enable type-safe test development with compile-time type checking.
- **R-TS-TEST-004** MAY: Test files MAY be co-located with source files in a `__tests__/` subdirectory as an alternative organization strategy, provided the `.test.ts` naming convention is maintained.
- **R-TS-TEST-005** MUST: Test files support both positive test cases (agent implementations) and negative test cases (e.g., invalid-agent.test.ts) for comprehensive test coverage.
- **R-TS-TEST-006** MUST: TypeScript compilation for tests succeed without type errors.
- **R-TS-TEST-007** MUST: Test runner configuration correctly identify and execute all `.test.ts` files with no orphaned or undiscovered tests.

### Verify

```bash
# Count test files matching .test.ts pattern
find tests/unit -name '*.test.ts' -type f | wc -l

# Verify test structure and content
grep -r "describe\|test\|it" tests/unit/**/*.test.ts | head -n 5

# Check TypeScript compilation for tests
npx tsc --noEmit --project tsconfig.test.json 2>&1 | grep -c 'error' || echo '0'
```

**Accept when:**
- All test files in `tests/unit/` directory use the `.test.ts` extension and are discoverable by the test runner
- TypeScript compilation succeeds for all test files without type errors
- Test runner configuration correctly identifies and executes all `.test.ts` files, with no orphaned or undiscovered tests
- Test files follow consistent naming convention across agent implementations, CLI handlers, and core business logic
