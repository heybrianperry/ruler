<rule_activation id="512029e2-b289-4555-91a1-7a426e2c690e" title="Standardize Environment Variable Configuration Management Across Agent Implementations" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent implementations and configuration management components within the system. All new agents and configuration loaders MUST comply with these rules from their initial implementation.
</rule_activation>

### Rules

- **R-ENV-001** MUST NOT: Agent implementations MUST NOT directly access process.env or equivalent environment variable APIs; all access MUST go through the configuration management layer (UnifiedConfigLoader).

### Verify

```bash
# Check for direct process.env access in agent implementations
grep -r 'process\.env' src/agents/ src/core/ --include='*.ts' | grep -v 'UnifiedConfigLoader' | wc -l | grep -q '^0$'

# Verify all agent files reference UnifiedConfigLoader
grep -r 'UnifiedConfigLoader' src/agents/ --include='*.ts' | wc -l | awk '{if($1>=8) exit 0; else exit 1}'

# Run configuration validation tests with coverage threshold
npm test -- --testPathPattern='config.*test' --coverage --coverageThreshold='{"global":{"branches":80,"functions":80,"lines":80}}'
```

**Accept when:**
- No direct process.env access exists in agent implementations outside of UnifiedConfigLoader
- All 8+ agent files reference UnifiedConfigLoader for configuration access
- Configuration validation tests achieve at least 80% code coverage and include failure scenarios

<enforcement>
Claude Code MUST verify all three conditions before accepting changes to agent implementations or configuration management components. Violations detected by automated static analysis MUST block CI pipeline and pull request merging.
</enforcement>
