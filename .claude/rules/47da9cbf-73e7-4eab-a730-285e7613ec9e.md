<rule_activation id="47da9cbf-73e7-4eab-a730-285e7613ec9e" title="Standardize Test File Organization in tests/ Directory with Unit and Integration Separation: Integration Tests Named" applies_to="**/*">
These rules are ALWAYS ACTIVE for all test file creation and organization within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** SHOULD: Integration tests SHOULD be named descriptively to indicate the integration scenario being tested (e.g., skills-mcp-agent-integration.test.ts, hierarchical-rules.test.ts)
- **R-TEST-002** MUST: All test files with .test.ts or .test.js extension MUST be located under tests/unit/ or tests/integration/ directories
- **R-TEST-003** MUST: Test runner MUST be able to execute unit tests and integration tests as separate suites
- **R-TEST-004** MUST: CI/CD pipeline MUST run unit tests independently from integration tests with appropriate timing thresholds

### Verify

```bash
# Check that all tests are properly organized in unit/integration directories
find tests/ -name '*.test.ts' -o -name '*.test.js' | grep -v -E '^tests/(unit|integration)/' && echo 'Found tests outside unit/integration dirs' || echo 'All tests properly organized'

# Verify required test directories exist
test -d tests/unit && test -d tests/integration && echo 'Required test directories exist' || echo 'Missing required test directories'

# Count unit tests
npm test -- --testPathPattern=tests/unit --listTests | wc -l
```

**Accept when:**
- All test files with .test.ts or .test.js extension are located under tests/unit/ or tests/integration/ directories
- Test runner can successfully execute unit tests and integration tests as separate suites
- CI/CD pipeline successfully runs unit tests independently from integration tests with appropriate timing thresholds
- Integration tests are named descriptively to indicate the integration scenario being tested

<enforcement>
Claude Code MUST NOT skip or defer verification. Automated CI checks MUST scan for test files outside the approved directory structure. Pull requests with incorrectly placed tests MUST be flagged for revision during code review.
</enforcement>
