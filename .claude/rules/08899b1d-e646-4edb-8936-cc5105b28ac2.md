<rule_activation id="08899b1d-e646-4edb-8936-cc5105b28ac2" title="Adopt Vitest with vi.mock() for Unit Testing TypeScript Agent Modules: Unit Tests Not" applies_to="tests/unit/**/*.test.ts">
These rules are ALWAYS ACTIVE for all unit test development in the codebase, particularly for TypeScript agent modules and CLI handlers.
</rule_activation>

### Rules

- **R-VITEST-001** MUST NOT: Unit tests MUST NOT depend on external services, file systems, or network resources without proper mocking.
- **R-VITEST-002** MUST: All unit test files in tests/unit/ MUST use Vitest imports (import { describe, test, expect, vi } from 'vitest').
- **R-VITEST-003** MUST: Use vi.mock('module-path') at the top of test files before imports to ensure proper hoisting.
- **R-VITEST-004** MUST: Implement beforeEach(() => vi.clearAllMocks()) in test suites to prevent mock state leakage between tests.
- **R-VITEST-005** MUST: Test files MUST follow naming convention *.test.ts and be located in tests/unit directory structure.
- **R-VITEST-006** SHOULD: For agent tests, mock external dependencies like file system, network calls, and third-party libraries consistently.
- **R-VITEST-007** SHOULD: Complement unit tests with integration tests that use real dependencies to mitigate over-mocking risks.
- **R-VITEST-008** MAY: Use vi.resetAllMocks() or vi.clearAllMocks() in afterEach hooks for additional isolation when needed.

### Verify

```bash
# Count vi.mock usage in unit tests
grep -r "vi.mock" tests/unit/ | wc -l

# Verify Vitest imports are present
grep -r "from 'vitest'" tests/unit/**/*.test.ts | wc -l

# Count total unit test files
find tests/unit -name '*.test.ts' -type f | wc -l

# Sample test structure verification
grep -r "describe\|test\|it" tests/unit/ | head -5

# Run Vitest suite
vitest run
```

**Accept when:**
- All unit test files in tests/unit/ use Vitest imports (import { describe, test, expect, vi } from 'vitest')
- At least 80% of agent test files use vi.mock() for dependency isolation
- Test files follow naming convention *.test.ts and are located in tests/unit directory structure
- CI pipeline successfully runs vitest command and all unit tests pass
- No external service, file system, or network calls are made without mocking
