<rule_activation id="acf762eb-2354-4403-8661-f21473554960" title="Adopt Vitest with vi.mock() for Unit Testing TypeScript Agent Modules: Test Files Organize" applies_to="tests/unit/**/*.test.ts">
These rules are ALWAYS ACTIVE for all unit test development in the codebase, particularly for TypeScript agent modules and CLI handlers in tests/unit directory.
</rule_activation>

### Rules

- **R-VITEST-001** SHOULD: Test files SHOULD organize tests using describe() blocks for logical grouping of related test cases.
- **R-VITEST-002** MUST: All unit test files in tests/unit/ MUST use Vitest imports (import { describe, test, expect, vi } from 'vitest').
- **R-VITEST-003** MUST: Use vi.mock('module-path') at the top of test files before imports to ensure proper hoisting.
- **R-VITEST-004** MUST: Implement beforeEach(() => vi.clearAllMocks()) in test suites to prevent mock state leakage between tests.
- **R-VITEST-005** SHOULD: Agent tests SHOULD mock external dependencies like file system, network calls, and third-party libraries consistently.
- **R-VITEST-006** MUST: Test files MUST follow naming convention *.test.ts and be located in tests/unit directory structure.
- **R-VITEST-007** SHOULD: At least 80% of agent test files SHOULD use vi.mock() for dependency isolation.

### Verify

```bash
# Count vi.mock usage across unit tests
grep -r "vi.mock" tests/unit/ | wc -l

# Verify Vitest imports are present
grep -r "from 'vitest'" tests/unit/**/*.test.ts | wc -l

# Count total test files
find tests/unit -name '*.test.ts' -type f | wc -l

# Sample test structure verification
grep -r "describe\|test\|it" tests/unit/ | head -5

# Run Vitest suite
vitest run
```

**Accept when:**
- All unit test files in tests/unit/ use Vitest imports (import { describe, test, expect, vi } from 'vitest')
- At least 80% of agent test files use vi.mock() for dependency isolation
- Test files follow naming convention *.test.ts and are located in tests/unit directory structure
- CI pipeline successfully runs vitest command and all unit tests pass
- describe() blocks are used for logical grouping of related test cases
- beforeEach/afterEach hooks include vi.clearAllMocks() or vi.resetAllMocks() for proper cleanup
