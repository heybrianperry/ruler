<rule_activation id="ba2b27c1-a71e-44f4-ae2f-1002212faeb9" title="Standardize Test File Organization in tests/ Directory with Unit and Integration Separation: Test Files Placed" applies_to="**/*">
These rules are ALWAYS ACTIVE for all test file creation and organization within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** MUST: All test files MUST be placed under a top-level tests/ directory
- **R-TEST-002** MUST: Test files with .test.ts or .test.js extension MUST be located under tests/unit/ or tests/integration/ directories
- **R-TEST-003** MUST: Unit tests for core engine components, agents, and utilities MUST be placed in tests/unit/
- **R-TEST-004** MUST: Integration tests for multi-component interactions MUST be placed in tests/integration/
- **R-TEST-005** SHOULD: Test runner configuration SHOULD support running unit and integration tests as separate suites
- **R-TEST-006** SHOULD: CI/CD pipeline SHOULD execute unit tests on every commit and integration tests on pull requests or specific branches
- **R-TEST-007** MAY: Test fixtures, mock data files, and test utilities MAY be organized in tests/helpers/ or tests/utils/

### Verify

```bash
# Check for test files outside unit/integration directories
find tests/ -name '*.test.ts' -o -name '*.test.js' | grep -v -E '^tests/(unit|integration)/' && echo 'Found tests outside unit/integration dirs' || echo 'All tests properly organized'

# Verify required test directories exist
test -d tests/unit && test -d tests/integration && echo 'Required test directories exist' || echo 'Missing required test directories'

# Count unit tests
npm test -- --testPathPattern=tests/unit --listTests | wc -l
```

**Accept when:**
- All test files with .test.ts or .test.js extension are located under tests/unit/ or tests/integration/ directories
- Test runner can successfully execute unit tests and integration tests as separate suites
- CI/CD pipeline successfully runs unit tests independently from integration tests with appropriate timing thresholds
- Automated CI checks confirm no test files exist outside the approved directory structure

<enforcement>
Claude Code MUST NOT skip or defer verification. All test files MUST comply with the directory structure defined in R-TEST-001 through R-TEST-004. Violations MUST be flagged during code review and CI pipeline execution MUST fail if test files are found outside tests/unit/ or tests/integration/ directories.
</enforcement>
