These rules are ALWAYS ACTIVE for all TypeScript test file development and test framework configuration.
</rule_activation>

### Rules

- **R-TS-001** MUST: Unit tests MUST be organized under the tests/unit/ directory, with subdirectories mirroring the source code structure (e.g., tests/unit/agents/ for agent tests, tests/unit/cli/ for CLI tests).
- **R-TS-002** MUST: All TypeScript unit test files MUST use the .test.ts naming convention for automatic test discovery.
- **R-TS-003** MUST: Test files MUST be written in TypeScript to enable compile-time type checking and type safety.
- **R-TS-004** SHOULD: Test runner configuration (Jest or Vitest) SHOULD be set up to automatically discover files matching the **/*.test.ts pattern in the tests/ directory.
- **R-TS-005** SHOULD: TypeScript compilation for tests SHOULD be configured with appropriate tsconfig.json settings, ensuring test files have access to necessary type definitions.
- **R-TS-006** MAY: Developers MAY request exceptions for specialized testing tools requiring alternative file extensions (e.g., .spec.ts) through the exception process.

### Verify

```bash
# Count all .test.ts files in tests/unit directory
find tests/unit -name '*.test.ts' -type f | wc -l

# Verify test structure contains describe/test/it blocks
grep -r "describe\|test\|it" tests/unit/**/*.test.ts | head -n 5

# Verify TypeScript compilation succeeds for all test files
npx tsc --noEmit --project tsconfig.test.json 2>&1 | grep -c 'error' || echo '0'
```

**Accept when:**
- All test files in tests/unit/ directory use the .test.ts extension and are discoverable by the test runner
- TypeScript compilation succeeds for all test files without type errors
- Test runner configuration correctly identifies and executes all .test.ts files, with no orphaned or undiscovered tests
- Directory structure mirrors source code organization (tests/unit/agents/, tests/unit/cli/, etc.)
