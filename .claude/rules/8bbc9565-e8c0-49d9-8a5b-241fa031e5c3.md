<rule_activation id="8bbc9565-e8c0-49d9-8a5b-241fa031e5c3" title="Standardize Test File Naming and Organization for CI/CD Integration: Test Files Use" applies_to="**/*.test.ts">
These rules are ALWAYS ACTIVE for all test file creation and CI/CD pipeline configuration activities.
</rule_activation>

### Rules

- **R-TEST-001** MAY: Test files MAY use descriptive kebab-case names for multi-word components (e.g., mcp-backup-prevention.test.ts)
- **R-TEST-002** MUST: All test files MUST use the .test.ts extension for automated test discovery
- **R-TEST-003** SHOULD: Test files SHOULD be organized into appropriate directories (unit/integration/e2e) based on their scope
- **R-TEST-004** MUST: Test files MUST be discoverable by the configured test runner using the glob pattern **/*.test.ts

### Verify

```bash
# Count test files with .test.ts extension
find tests -name '*.test.ts' | wc -l

# Verify test file structure and content
grep -r 'describe\|test\|it' tests/ --include='*.test.ts' | head -n 5

# Check test directory organization
ls -la tests/unit tests/integration tests/e2e 2>/dev/null || echo 'Test directories not found'

# Verify glob pattern matches all test files
find . -name '*.test.ts' -type f
```

**Accept when:**
- All test files use the .test.ts extension and are discoverable by the configured test runner
- Test files are organized into appropriate directories (unit/integration/e2e) based on their scope
- CI/CD pipeline successfully discovers and executes all tests without manual configuration for new test files
- Test coverage reports accurately reflect all test files in the project
- No test files exist outside the expected directory structure or with non-standard naming patterns

<enforcement>
Claude Code MUST NOT skip or defer verification. All test files MUST conform to the naming and organization standards before code is committed. Pre-commit hooks and CI/CD pipeline checks MUST validate compliance.
</enforcement>
