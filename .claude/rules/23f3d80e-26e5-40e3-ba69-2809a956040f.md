<rule_activation id="23f3d80e-26e5-40e3-ba69-2809a956040f" title="Standardize Environment Variable Configuration Management Across Agent Implementations: Configuration Values Containing" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent implementations, core configuration components, MCP propagation components, and test environments that simulate production configuration.
</rule_activation>

### Rules

- **R-CONFIG-001** MUST: Configuration values containing secrets (API keys, tokens, credentials) MUST NOT be logged or exposed in error messages.
- **R-CONFIG-002** MUST: All agent implementations (ZedAgent, GeminiCliAgent, RooCodeAgent, OpenCodeAgent, QwenCodeAgent, and future agents) MUST use UnifiedConfigLoader for configuration access.
- **R-CONFIG-003** MUST: No direct process.env access is permitted in agent implementations outside of UnifiedConfigLoader.
- **R-CONFIG-004** SHOULD: Configuration SHOULD be validated eagerly at startup with fail-fast behavior rather than lazy loading.
- **R-CONFIG-005** SHOULD: Configuration state SHOULD be immutable to prevent concurrency bugs in multi-threaded or async execution contexts.

### Verify

```bash
# Verify no direct process.env access in agent implementations
grep -r 'process\.env' src/agents/ src/core/ --include='*.ts' | grep -v 'UnifiedConfigLoader' | wc -l | grep -q '^0$'

# Verify all 8+ agent files reference UnifiedConfigLoader
grep -r 'UnifiedConfigLoader' src/agents/ --include='*.ts' | wc -l | awk '{if($1>=8) exit 0; else exit 1}'

# Verify configuration validation tests achieve 80% coverage
npm test -- --testPathPattern='config.*test' --coverage --coverageThreshold='{"global":{"branches":80,"functions":80,"lines":80}}'
```

**Accept when:**
- No direct process.env access exists in agent implementations outside of UnifiedConfigLoader
- All 8+ agent files reference UnifiedConfigLoader for configuration access
- Configuration validation tests achieve at least 80% code coverage and include failure scenarios
- All secrets are properly masked in logs and error messages
- Configuration validation catches common misconfiguration scenarios (missing required vars, invalid formats)

<enforcement>
Claude Code MUST NOT skip or defer verification. All three verify commands MUST pass before accepting changes to agent implementations or configuration management components.
</enforcement>
