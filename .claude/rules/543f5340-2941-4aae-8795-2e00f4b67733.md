These rules are ALWAYS ACTIVE for all test files within the CI/CD pipeline and apply to all agent-related test suites, configuration validation tests, and integration tests.
</rule_activation>

### Rules

- **R-TEST-001** MUST: All agent implementations MUST have corresponding unit test files following the naming pattern `tests/unit/agents/{AgentName}.test.ts`
- **R-TEST-002** MUST: Configuration validation tests MUST exist for all security-critical input fields (type, HTTP config, MCP settings)
- **R-TEST-003** MUST: Test organization MUST follow the three-tier structure: unit tests, integration tests, and configuration tests with clear naming conventions
- **R-TEST-004** SHOULD: Use test file templates that include standard sections for input validation, configuration override testing, and nested behavior validation
- **R-TEST-005** SHOULD: Implement test helpers for common validation patterns (MCP config validation, HTTP type checking, agent-specific disable logic) to reduce duplication

### Verify

```bash
# Count test files matching expected patterns
find tests/ -name '*.test.ts' | grep -E '(unit/agents/|integration/|claude-|gemini-|mcp-)' | wc -l

# Verify agent test suites exist
grep -r 'describe.*Agent' tests/unit/agents/ | wc -l

# Verify security-critical validation coverage
grep -r 'input.*validation\|type.*field\|MCP.*config' tests/ | wc -l
```

**Accept when:**
- All agent implementations have corresponding test files in `tests/unit/agents/` directory
- Configuration validation tests exist for all security-critical input fields (type, HTTP config, MCP settings)
- Test organization follows the three-tier structure: unit tests, integration tests, and configuration tests with clear naming conventions
- Test file naming conventions are consistent across all agent types
- CI/CD pipeline test execution passes with mandatory requirements met

Claude Code MUST NOT skip or defer verification. All agent implementations MUST have corresponding test files and configuration validation tests before code is approved for merge. Violations block CI/CD pipeline execution.
