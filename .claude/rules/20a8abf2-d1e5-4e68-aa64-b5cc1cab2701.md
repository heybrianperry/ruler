<rule_activation id="20a8abf2-d1e5-4e68-aa64-b5cc1cab2701" title="Standardize on Vitest for Unit Testing Agent Modules: Agent Tests Follow" applies_to="tests/unit/agents/**/*.test.ts">
These rules are ALWAYS ACTIVE for all agent module unit tests in tests/unit/agents/. This ADR standardizes testing practices across all agent implementations (OpenCodeAgent, WarpAgent, WindsurfAgent, JunieAgent, JulesAgent) to ensure consistent patterns, shared utilities, and predictable developer experience.
</rule_activation>

### Rules

- **R-VITEST-001** SHOULD: Agent tests SHOULD follow consistent describe/it block structure for test organization.

### Verify

```bash
# Verify all agent test files import from vitest
grep -r "from 'vitest'" tests/unit/agents/*.test.ts | wc -l

# Count total agent test files
find tests/unit/agents -name '*.test.ts' -type f | wc -l

# Run all agent tests with vitest
npx vitest run tests/unit/agents/ --reporter=verbose
```

**Accept when:**
- All agent test files in tests/unit/agents/ import test functions from 'vitest' package
- Test files follow naming convention {AgentName}.test.ts and execute successfully with vitest runner
- No agent unit tests use alternative testing frameworks (jest, mocha, ava) in the tests/unit/agents/ directory
- Vitest is installed as a dev dependency with appropriate TypeScript configuration in vitest.config.ts
- Shared test utilities exist in tests/unit/agents/helpers/ for common agent mocking and setup patterns
- CI pipeline runs vitest with coverage reporting and enforces coverage threshold violations

<enforcement>
Claude Code MUST verify all agent test files use Vitest with consistent describe/it block structure. Verification is mandatory before accepting agent test implementations. CI pipeline MUST fail if agent tests are found using non-Vitest frameworks or violate naming conventions.
</enforcement>
