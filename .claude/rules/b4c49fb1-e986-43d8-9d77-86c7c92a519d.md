<rule_activation id="b4c49fb1-e986-43d8-9d77-86c7c92a519d" title="Adopt Unified Configuration Loading Pattern for Runtime Environment Management: Configuration Objects Immutable" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent implementations and configuration management components within the system. All new agents and configuration loaders MUST comply with these rules.
</rule_activation>

### Rules

- **R-CFG-001** SHOULD: Configuration objects SHOULD be immutable after loading to prevent unexpected state changes during runtime.

### Verify

```bash
# Verify UnifiedConfigLoader usage across agent implementations
grep -r "UnifiedConfigLoader" src/agents/ | wc -l

# Verify imports of UnifiedConfigLoader in TypeScript/JavaScript files
grep -r "import.*UnifiedConfigLoader" src/ --include="*.ts" --include="*.js"

# Verify configuration loading tests pass
npm test -- --grep "config.*load" 2>&1 | grep -E "(passing|failing)"
```

**Accept when:**
- All agent implementations (ZedAgent, GeminiCliAgent, RooCodeAgent, OpenCodeAgent, QwenCodeAgent) import and use UnifiedConfigLoader for configuration loading
- Configuration loading tests pass successfully, including tests for TOML parsing, environment variable precedence, and validation
- No direct file system reads for configuration files exist outside of UnifiedConfigLoader implementation
- Configuration objects are verified to be immutable after loading through code review and/or static analysis

<enforcement>
Claude Code MUST NOT skip or defer verification. All configuration loading patterns MUST use UnifiedConfigLoader and configuration objects MUST be immutable after loading. Violations are rejected during code review with feedback to use the standard pattern.
</enforcement>
