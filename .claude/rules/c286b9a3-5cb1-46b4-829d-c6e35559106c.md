<rule_activation id="c286b9a3-5cb1-46b4-829d-c6e35559106c" title="Standardize on Vitest for Unit Testing Agent Modules: Shared Test Utilities" applies_to="tests/unit/agents/**/*.test.ts">
These rules are ALWAYS ACTIVE for all agent module unit tests in tests/unit/agents/. This ADR standardizes on Vitest as the exclusive testing framework for all agent implementations and requires extraction of shared test utilities to common helper modules.
</rule_activation>

### Rules

- **R-VITEST-001** SHOULD: Shared test utilities and mocks for agent testing SHOULD be extracted to common test helper modules in tests/unit/agents/helpers/
- **R-VITEST-002** MUST: All agent unit test files in tests/unit/agents/ MUST import test functions from the 'vitest' package
- **R-VITEST-003** MUST: Agent test files MUST follow the naming convention {AgentName}.test.ts
- **R-VITEST-004** MUST: No agent unit tests in tests/unit/agents/ MAY use alternative testing frameworks (jest, mocha, ava)
- **R-VITEST-005** SHOULD: vitest SHOULD be installed as a dev dependency with TypeScript configuration in vitest.config.ts
- **R-VITEST-006** SHOULD: CI pipeline SHOULD run vitest with coverage reporting and fail builds on coverage threshold violations

### Verify

```bash
# Verify all agent test files import from vitest
grep -r "from 'vitest'" tests/unit/agents/*.test.ts | wc -l

# Count total agent test files
find tests/unit/agents -name '*.test.ts' -type f | wc -l

# Run all agent tests with vitest
npx vitest run tests/unit/agents/ --reporter=verbose

# Verify no alternative frameworks are used
grep -r "from 'jest'\|from 'mocha'\|from 'ava'" tests/unit/agents/*.test.ts | wc -l
```

**Accept when:**
- All agent test files in tests/unit/agents/ import test functions from 'vitest' package
- Test files follow naming convention {AgentName}.test.ts and execute successfully with vitest runner
- No agent unit tests use alternative testing frameworks (jest, mocha, ava) in the tests/unit/agents/ directory
- Shared test utilities are extracted to tests/unit/agents/helpers/ for common agent mocking and setup patterns
- CI pipeline is configured to run vitest with coverage reporting

<enforcement>
Claude Code MUST verify all rules in this activation block. Violations of R-VITEST-002, R-VITEST-003, and R-VITEST-004 (MUST-level rules) MUST block acceptance. SHOULD-level rules SHOULD be addressed but may have documented exceptions approved by engineering lead. Verification is mandatory before accepting any changes to agent test files.
</enforcement>
