<rule_activation id="cb05bd9d-08fe-4d00-8146-ff2e4e91105e" title="Standardize Test File Naming and Organization for CI/CD Integration: Test File Names" applies_to="tests/**/*.ts">
These rules are ALWAYS ACTIVE for all test file creation and CI/CD pipeline configuration activities.
</rule_activation>

### Rules

- **R-TEST-001** MUST: Test file names MUST clearly indicate the component or feature being tested, following the pattern [component-name].test.ts or [feature-name].test.ts
- **R-TEST-002** MUST: All TypeScript test files in the project MUST use the .test.ts extension
- **R-TEST-003** MUST: Test files MUST be organized into appropriate directories (unit/integration/e2e) based on their scope
- **R-TEST-004** MUST: CI/CD pipeline configurations MUST use the glob pattern **/*.test.ts for test discovery
- **R-TEST-005** SHOULD: Implement automated linting rules and pre-commit hooks to enforce test file naming patterns
- **R-TEST-006** SHOULD: Configure test runners (Jest, Vitest, etc.) to use the glob pattern **/*.test.ts for test discovery
- **R-TEST-007** SHOULD: Document the test organization structure in the project README and contributing guidelines

### Verify

```bash
# Count test files with .test.ts extension
find tests -name '*.test.ts' | wc -l

# Verify test file structure and content
grep -r 'describe\|test\|it' tests/ --include='*.test.ts' | head -n 5

# Check test directory organization
ls -la tests/unit tests/integration tests/e2e 2>/dev/null || echo 'Test directories not found'
```

**Accept when:**
- All test files use the .test.ts extension and are discoverable by the configured test runner
- Test files are organized into appropriate directories (unit/integration/e2e) based on their scope
- CI/CD pipeline successfully discovers and executes all tests without manual configuration for new test files
- Test coverage reports accurately reflect all test files in the project

<enforcement>
Claude Code MUST NOT skip or defer verification of test file naming patterns and organization. Violations MUST be flagged during code review and pre-commit checks MUST block commits with incorrectly named or located test files.
</enforcement>
