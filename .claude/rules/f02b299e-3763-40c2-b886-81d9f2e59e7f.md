These rules are ALWAYS ACTIVE for all agent component development and testing activities.
</rule_activation>

### Rules

- **R-AGENT-001** MUST: All agent component implementations MUST have corresponding unit test files located in tests/unit/agents/ directory

### Verify

```bash
# Verify agent test directory exists
test -d tests/unit/agents && echo 'Agent test directory exists' || echo 'Missing agent test directory'

# Count agent test files
find tests/unit/agents -name '*.test.ts' | wc -l

# List all agent test files
ls tests/unit/agents/*.test.ts 2>/dev/null || echo 'No agent tests found'
```

**Accept when:**
- The tests/unit/agents/ directory exists and contains .test.ts files
- Each agent implementation class has a corresponding {AgentName}.test.ts file in tests/unit/agents/
- Test files can be discovered and executed by the project's test runner without additional configuration
- CI pipeline validation confirms all agent classes have corresponding unit test files

<enforcement>
Claude Code MUST verify agent test structure compliance before approving agent component implementations. Violations must be flagged and remediation required before merge.
</enforcement>
