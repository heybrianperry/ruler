### Rules

- **R-VITEST-001** MUST: All agent module unit tests MUST use Vitest as the testing framework.
- **R-VITEST-002** MUST: All unit tests for agent modules in tests/unit/agents/ MUST import test functions from the 'vitest' package.
- **R-VITEST-003** MUST: New agent implementations added to the codebase MUST use Vitest for their unit tests.
- **R-VITEST-004** MUST: Refactored or updated existing agent test suites MUST use Vitest.
- **R-VITEST-005** MUST: Test utilities and helpers specifically for agent testing MUST use Vitest.
- **R-VITEST-006** MUST: Agent test files MUST follow the naming convention {AgentName}.test.ts.

### Verify

```bash
# Verify all agent test files import from vitest
grep -r "from 'vitest'" tests/unit/agents/*.test.ts | wc -l

# Count total agent test files
find tests/unit/agents -name '*.test.ts' -type f | wc -l

# Run all agent tests with vitest
npx vitest run tests/unit/agents/ --reporter=verbose

# Verify no alternative frameworks are used in agent tests
grep -r "from 'jest'\|from 'mocha'\|from 'ava'" tests/unit/agents/*.test.ts | wc -l
```

**Accept when:**
- All agent test files in tests/unit/agents/ import test functions from 'vitest' package
- Test files follow naming convention {AgentName}.test.ts and execute successfully with vitest runner
- No agent unit tests use alternative testing frameworks (jest, mocha, ava) in the tests/unit/agents/ directory
- vitest is installed as a dev dependency with appropriate TypeScript configuration in vitest.config.ts
