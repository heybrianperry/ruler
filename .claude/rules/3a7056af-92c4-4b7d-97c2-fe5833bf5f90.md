<rule_activation id="3a7056af-92c4-4b7d-97c2-fe5833bf5f90" title="Standardize Public API Export Patterns for Agent Integration: Integration Modules Mcp" applies_to="**/*">
These rules are ALWAYS ACTIVE for all agent implementations and public API modules. All new agent integrations and external API surfaces MUST comply with these export patterns.
</rule_activation>

### Rules

- **R-API-001** SHOULD: Integration modules (e.g., MCP merge utilities) SHOULD expose composable functions that can be combined for complex workflows.
- **R-API-002** MUST: All agent implementation files in src/agents/ MUST use named exports, not default exports (except during migration period per EXC-001).
- **R-API-003** MUST: All agent classes MUST follow the naming convention with 'Agent' suffix.
- **R-API-004** MUST: Central index file (src/agents/index.ts) MUST contain exports for all agent implementations.
- **R-API-005** SHOULD: Use TypeScript's 'export type' and 'export interface' to clearly separate type exports from value exports.
- **R-API-006** SHOULD: Consider using barrel exports (index.ts files) at the agent directory level to simplify imports for consumers.
- **R-API-007** MAY: Legacy agent implementations may temporarily use default exports during migration period (EXC-001).
- **R-API-008** MAY: Experimental or prototype agents in development may defer index registration (EXC-002).

### Verify

```bash
# Verify no default exports in agent implementation files (excluding test files)
grep -r 'export default' src/agents/*.ts | grep -v '.test.ts' | wc -l | grep -q '^0$'

# Verify all agent classes follow naming convention with 'Agent' suffix
grep -r 'export class.*Agent' src/agents/*.ts | wc -l

# Verify central index file exists and contains agent exports
test -f src/agents/index.ts && grep -q 'export.*Agent' src/agents/index.ts
```

**Accept when:**
- No default exports found in agent implementation files (excluding test files)
- All agent classes follow the naming convention with 'Agent' suffix
- Central index file exists and contains exports for all agent implementations
- All core utility modules in src/core/ expose public functionality through named exports
- All integration modules in src/mcp/ expose composable functions
- TypeScript compiler checks pass with no interface compliance violations
