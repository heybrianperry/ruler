These rules are ALWAYS ACTIVE for all agent component development and testing activities within the codebase.
</rule_activation>

### Rules

- **R-TEST-001** SHOULD: CLI handlers and command-line interface components SHOULD have dedicated unit tests in `tests/unit/cli/` subdirectory.
- **R-TEST-002** SHOULD: All agent component implementations (AmpAgent, WarpAgent, JulesAgent, WindsurfAgent, OpenCodeAgent, JunieAgent, and future agents) SHOULD follow standardized unit testing patterns.
- **R-TEST-003** SHOULD: Test files SHOULD follow the naming pattern `*.test.ts` and be located in appropriate subdirectories under `tests/unit/`.
- **R-TEST-004** SHOULD: Unit tests SHOULD cover error handling and invalid input scenarios for agent components.
- **R-TEST-005** SHOULD: Code coverage SHOULD maintain a minimum threshold of 80% across agent components.

### Verify

```bash
# Verify test file naming and location conventions
find tests/unit -name '*.test.ts' | grep -E 'tests/unit/(agents|cli)/.+\.test\.ts$'

# Verify code coverage meets minimum threshold
npm test -- --coverage --coverageThreshold='{"global":{"lines":80}}'

# Verify minimum number of agent test files exist
ls tests/unit/agents/*.test.ts | wc -l | grep -E '^[6-9]|^[1-9][0-9]+$'
```

**Accept when:**
- All test files follow the naming pattern `*.test.ts` and are located in appropriate subdirectories under `tests/unit/`
- Test suite executes successfully with at least 80% code coverage across agent components
- At least 6 agent test files exist in `tests/unit/agents/` directory, maintaining the established pattern
- CLI handler tests are present in `tests/unit/cli/` subdirectory for command-line interface components

<enforcement>
Claude Code MUST NOT skip or defer verification of these rules. All pull requests adding new agents or CLI handlers without corresponding unit tests in the correct location MUST fail CI checks and be blocked from merging.
</enforcement>
